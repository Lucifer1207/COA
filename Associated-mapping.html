ASSOCIATIVE  MAPPING

**** This  Technique  Overcome  The  Disadvantage
of  Direct  Mapping  by "  Permitting  Each  Main  Memory  Block  To
Be  Loaded  into  Any  line  Of  Cache ".****


**** It  Helps  Us  Make  A  Associative
Mapping  Comparatively  More  Flexible  Than  Direct  Mapping. ****


**** In  The  Case  Of  Associative  Mapping,
> The  Replacement  Algorithm  Is  Always  Required.

> The  Replacement  Algorithm  Suggests  A  Block  That  Is  To  Be  Replaced
Whenever  All  The  Cache  Lines  Happen  To  Be  Occupied.

> Replacement  Algorithms  Such  As  LRU
Algorithm  ,  FCFS  Algorithm , Etc.,  Are  Employed****

For  Example,

* Let  Us  Consider  The  Scenario  Given  As  Follows :






fig :- ASSOCIATIVE  MAPPING





 Here,  We  Can  See  That ,

> Every  Single  Line  Of  Cache  Is  Available  Freely.
> Thus  ,  Any  Main  Memory  Block  Can  Map  To  A  Line  Of  The  Cache.
> In  Case  All  The  Cache  Lines  Are  Occupied,  One  Of  The  blocks  That  Exists  Already  Needs  To  Be  Replaced.






fig :-PHYSICAL  ADDRESS





ADVANTAGES  &  DISADVANTAGES

ADVANTAGES

 1. Fast.

2. There Will  Be  No  Placement  Restrictions  To   Map  The  Main  Memory  Block  With
         Cache  memory  Line.  The  Main  Memory  Block  Can  Map  With  Any
                       Block  that  present  in  the  cache  memory.

3. Easy  To  Work  And  Implement.

DISADVANTAGES

 1. Expensive .

2. Tag  Field  Is  Long .

3.  Uses  Extra  Space .
