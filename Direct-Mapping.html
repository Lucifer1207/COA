DIRECT  MAPPING

It  Is  A  Procedure  Used  To  Assign  Each
 Memory  Block  In  The  Main  Memory  To  A  Particular  Line  In  The  Cache.


If  A  Line  Is  Already  Filled  With
 A  Memory  Block  And  A  New  Block Needs  To  Be  Loaded , Then
 The  Old  Block   Is  Discarded  From  The  Cache.


****In The Case Of Direct Mapping,
> There  Is  No  Requirement  For  A  Replacement  Algorithm.

> It  Is  Because  The  Block  Of  The  Main  Memory  Would  Be  Able  To  mMp
 To  A  Certain  Line  Of  The  Cache  Only.

> Thus,  The  Incoming (New)  Block  Always
 Happens  To  Replace  The  Block  That  Already Exists,  If  Any,
 In  This  Certain  Line.****

Cache  Line  Number (i)=
Address  Of  The  Main  Memory  Block(j)
Modulo(%)
Total  Number  Of  Lines  In  Cache(n)


i = J mod n

For  Example,

* Let  Us  Consider  That  Particular  Cache  Memory  Is   Divided  Into  A  total  Of  ‘n’  Number  Of  Lines.
* Then,  the  Block  ‘j’ Of  The  Main  Memory  Would  Be  Able  To  Map  To  Line  Number  Only  Of  The  Cache  (j mod n).

ADVANTAGES  &  DISADVANTAGES

ADVANTAGES

 1. Simple  And  Cheap.
2. Access  Is  Very  Fast.
3. Tag  Field  Is  Short.

DISADVANTAGES

 > Can  Produce  A  Low  Hit  Ratio  Even  If  Only  A  Very  Small  Part  Of  Cache  Is  Effective  Used.
